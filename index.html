pip3 install requests colorama python-nmap scapy # port_scanner.py import socket, concurrent.futures, sys, argparse from colorama import Fore, init init() def scan(host, port): try: s = socket.socket() s.settimeout(0.5) s.connect((host, port)) try: banner = s.recv(1024).decode().strip() except: banner = "" s.close() return port, True, banner except: return port, False, "" parser = argparse.ArgumentParser() parser.add_argument("host") parser.add_argument("-p", default="1-1024") args = parser.parse_args() start, end = map(int, args.p.split("-")) with concurrent.futures.ThreadPoolExecutor(200) as ex: results = list(ex.map(lambda p: scan(args.host, p), range(start, end+1))) for port, open, banner in sorted(results): if open: print(f"{Fore.GREEN}[+] {port}/tcp OPEN {banner}{Fore.RESET}")

Подними ELK через Docker. Настрой Filebeat для отправки /var/log/auth.log в Elasticsearch. Создай Kibana Dashboard: топ-10 IP по неудачным логинам + график по времени. Сохрани скриншот дашборда.
ELK работает, логи поступают реальном времени. Dashboard визуализирует аномалии.

Задание
Напиши 3 Sigma-правила: SSH брутфорс, сканирование портов (Nmap), попытка sudo от неизвестного пользователя. Конвертируй в Elasticsearch DSL. Настрой алерты в Kibana. Для каждого: привяжи к технике MITRE ATT&CK.
3 правила + алерты в Kibana. Понимаешь матрицу MITRE ATT&CK и можешь найти технику по TTP.
Нед. 24 Финальный проект: полный пентест HackTheBox Easy Портфолио ▶
Зачем: профессиональный пентест-отчёт — главный артефакт портфолио. Это то, что просят показать на 90% собеседований в кибербезе.
  • 🔗 HackTheBox: Easy retired machines
  • 🔗 TCM Security: шаблон отчёта (GitHub)
  • 🔗 VulnHub: Easy machines как альтернатива
Структура отчёта
1. Executive Summary (для нетехнического читателя) 2. Scope & Methodology 3. Findings (для каждой уязвимости): - Title + CVSS Score - Affected Component - Description - Steps to Reproduce - Evidence (скриншоты) - Remediation Recommendation 4. Appendix (инструменты, команды)
Задание
Пентест одной Easy-машины на HTB (retired). Отчёт по шаблону TCM Security: минимум 3 уязвимости, скриншоты каждого шага, CVSS оценки, рекомендации. Экспортируй в PDF. Загрузи анонимизированную версию на GitHub.
Профессиональный PDF-отчёт в портфолио. Это твой главный документ на собеседование.

🚀 Первые 3 действия — прямо сегодня
1
Установи лабораторию (1–2 часа)
Скачай VirtualBox (бесплатно, virtualbox.org). Скачай Kali Linux ISO (kali.org/get-kali). Создай VM: 4 GB RAM, 2 CPU, 50 GB диск. Это твоя изолированная песочница на все 6 месяцев — без неё нет практики.

sudo apt update && sudo apt full-upgrade -y
2
Зарегистрируйся на TryHackMe + начни Pre-Security Path (15 минут)
Создай аккаунт на tryhackme.com. Сразу открой Pre-Security Path — он бесплатный и идеально покрывает Неделю 1. Настрой публичный профиль: работодатели смотрят на статистику THM.

Прямая ссылка: tryhackme.com/path/outline/presecurity
3
Создай GitHub для портфолио (30 минут)
Если нет аккаунта — создай на github.com. Создай репозиторий cybersec-journey. Добавь папки: /notes, /tools, /ctf-writeups, /pentest-reports. Напиши README: кто ты, что изучаешь, цель через 6 месяцев. Это твой дневник обучения — работодатели ценят консистентность и публичность.

Через 6 месяцев этот репозиторий станет твоим главным активом на собеседовании.
Главный совет: Самая частая ошибка начинающих — бесконечная теория без практики. После каждой недели у тебя должен быть конкретный артефакт: скрипт, отчёт, writeup, конфиг. Работодатель не спросит «что ты читал» — он спросит «покажи что ты сделал». Твой GitHub — это и есть твоё резюме.

Твой бэкграунд программиста — огромный плюс перед людьми без IT-опыта. Python-скрипты, понимание логики кода, умение читать исходники — это то, что отличает хорошего пентестера от среднего. Используй это с первого дня.
